# アプリ全体の設計を理解しよう

---

## このドキュメントで学べること

「なぜこのファイル構成になっているのか」「データはどこを通って画面に届くのか」を理解します。
設計の理由がわかると、新しい機能を追加するときにどこに何を書けばいいかが自然とわかるようになります。

---

## 使っている技術とその理由

### Next.js（App Router）
**ひとことで言うと**: Reactでウェブアプリを作るための「フレームワーク（便利な道具箱）」です。

普通の React だけだと「ページ遷移の仕組み」「サーバーとの通信」などを自分で全部作らないといけません。
Next.js を使うと、決められた場所にファイルを置くだけで自動的にページが作られます。

**App Router とは**: Next.js の新しいページの作り方のルール。`page.tsx` というファイルを作るだけで、そのフォルダ名がそのままURLになります。例えば `app/board/page.tsx` を作ると `/board` というページが自動で作られます。

**なぜ使うか**: 日本のWeb開発案件の多くがReact/Next.jsを採用しており、実務で最もよく使われる技術だから。

---

### Supabase
**ひとことで言うと**: データベース・ログイン機能・セキュリティがセットになったクラウドサービスです。

本来、ユーザー情報を保存したり、ログイン機能を作るには「バックエンドサーバー」を別途用意する必要があります。Supabase を使うと、それを自分で作らなくても済みます。

**なぜ使うか**:
- 無料から始められる
- 「どのユーザーがどのデータを見られるか」をDBレベルで制御できる（RLS という機能）
- 本格的なデータベース（PostgreSQL）の知識が身につく

---

### Anthropic Claude API
**ひとことで言うと**: このアプリの「AI機能」を動かしているサービスです。

案件の自動生成・クライアントとのチャット・コードレビューはすべて Claude（AI）が行っています。日本語の精度が高く、決まった形式（JSON）でデータを返すのが得意なので採用しました。

## フォルダ構成の意味

```
app/           ← 「どのURLでどのページを表示するか」を管理する場所
components/    ← 画面のパーツ（ボタン、カードなど）を置く場所
lib/           ← データベース接続やAI呼び出しなどの「裏方の処理」を置く場所
types/         ← データの形（型）を定義する場所
hooks/         ← 画面の状態管理をまとめる場所
docs/          ← このドキュメント群
supabase/      ← データベースの設計ファイルを置く場所
```

**ポイント**: `components/` の中をさらに `board/`（案件ボード関連）、`workspace/`（作業画面関連）などのフォルダに分けています。こうすることで「このUIはどの機能のものか」がファイル名を見るだけでわかります。実務でよく使われる整理の仕方です。

---

## データの流れ（ユーザーが操作してから画面が変わるまで）

難しく見えますが、大きな流れは「ユーザーが操作 → サーバーが処理 → 結果を画面に反映」です。

```
① ユーザーがボタンを押す
    ↓
② ブラウザ上のコンポーネントが API に「お願い」を送る
    ↓
③ サーバー（API Route）が受け取り、認証チェックをする
    ↓
④ Supabase（データベース）にデータを保存・取得する
    ↓
⑤ 必要なら Claude AI にリクエストを送る
    ↓
⑥ 結果をブラウザに返す
    ↓
⑦ 画面が更新される
```

---

## 「ブラウザ用」と「サーバー用」のクライアントの使い分け

`lib/supabase/` の中に `client.ts` と `server.ts` の2つのファイルがあります。同じSupabaseなのになぜ2つあるかというと、**ログイン情報の管理方法が違うから**です。

- **client.ts（ブラウザ用）**: `'use client'` と書かれたコンポーネントで使う。ブラウザが自動でログイン情報を管理してくれる。
- **server.ts（サーバー用）**: API Route や Server Component で使う。サーバー側でリクエストに含まれるCookieを手動で読み書きしてログイン情報を確認する。

---

## よくあるミスと対処法

1. **ボタンが動かない** → ボタンや入力欄など「ユーザーが操作するもの」を含むコンポーネントには、ファイルの先頭に `'use client'` を書く必要がある
2. **データが取れない** → Server Component でCookieを取得するときは `await cookies()` のように `await` が必要（Next.js 15以降）
3. **TypeScriptのエラーが出る** → Supabaseから取得したデータの型が `any` になる場合は、`types/index.ts` に自分で型を定義して使う
