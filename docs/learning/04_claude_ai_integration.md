# Claude AIとの連携を理解しよう

---

## AIに「お願い」を送る基本の形

このアプリではAIに文章を生成してもらうために Anthropic SDK を使っています。基本的な使い方は以下のとおりです。

```typescript
import Anthropic from '@anthropic-ai/sdk'

const anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY })

const message = await anthropic.messages.create({
  model: 'claude-haiku-4-5-20251001', // 使うAIモデル
  max_tokens: 2048,                   // 返答の最大文字数（トークン数）
  messages: [
    { role: 'user', content: 'こんにちは' }
  ],
})

const text = message.content[0].text  // AIの返答テキストを取り出す
```

**`max_tokens` とは**: AIが返答できる文章の最大量です。小さすぎると返答の途中で切れてしまいます。1トークンはだいたい1〜2文字に相当します。

## このアプリの3つのAI機能

### 1. 案件生成（lib/ai/project-generator.ts）

「ジュニア向けのLP制作案件を作って」というお願いをAIに送り、案件の内容（タイトル・要件定義・報酬・使用技術・クライアント設定）をJSON形式で返してもらいます。

**工夫したプロンプト（AIへの指示文）**:
```
「以下のJSON形式のみで回答してください（マークダウンコードブロック不要、JSONのみ）」
```

AIはデフォルトで返答をコードブロック（\`\`\`json ... \`\`\`）で囲むことが多いです。しかしこのアプリは `JSON.parse()` でそのまま解析するため、余分な記号があるとエラーになります。「JSONだけ返して」と明示的に指示することで解決しています。

**使うモデル**: `claude-haiku-4-5-20251001`（高速・軽量モデル）
→ 案件の文章生成は Haiku でも十分な品質が出るため。速度とコストを優先。

---

### 2. ヒアリングチャット（lib/ai/chat-client.ts）

AIに「あなたは〇〇会社の△△というクライアントです」という役を与えて、ユーザーとの会話を成立させます。

**システムプロンプトとは**: 会話全体に適用される「AIの役割設定」です。毎回のメッセージに含まれるわけではなく、「あなたはこういうキャラクターです」という大前提として機能します。

```typescript
system: `あなたは「${persona.name}」という架空のクライアントです。
会社: ${persona.company}
性格: ${persona.personality}
口調: ${persona.tone}`
```

**会話履歴をすべて送る理由**: AIは「記憶」を持っていません。毎回の会話で「今までどんなやり取りをしたか」を全部送ることで、前の内容を踏まえた返答ができるようになります。

```typescript
// データベースのメッセージをAI用の形式に変換して全部渡す
const anthropicMessages = messages.map(msg => ({
  role: msg.role === 'user' ? 'user' : 'assistant',
  content: msg.content,
}))
```

**使うモデル**: `claude-haiku-4-5-20251001`（高速・軽量モデル）
→ チャットは返答速度が最重要。品質よりスピードを優先。

---

### 3. コードレビュー（lib/ai/reviewer.ts）

提出されたコードを「可読性・セキュリティ・機能性」の3つの観点で評価してもらいます。

**評価の観点を具体的に指示する理由**: 「コードをレビューして」と言うだけでは、AIが何を重視するかがバラバラになります。「可読性は変数名が適切かどうかを見て」「セキュリティはXSSやSQLインジェクションがないかを確認して」と具体的に伝えることで、一定の基準でスコアリングされます。

**使うモデル**: `claude-haiku-4-5-20251001`
→ コストと速度のバランスを考慮。

## AIのモデルの選び方

Claude には複数のモデルがあり、用途によって使い分けます。

| モデル | 特徴 | 向いている用途 |
|---|---|---|
| `claude-opus-4-6` | 最高品質・低速・高コスト | 複雑な推論・高精度な分析が必要な場面 |
| `claude-sonnet-4-6` | バランス型 | 品質と速度のバランスが必要な場面 |
| `claude-haiku-4-5-20251001` | 高速・軽量・低コスト | リアルタイムなチャット・シンプルな文章生成 |

このアプリでは全機能に Haiku を採用しています（速度・コストを優先）。

## JSONパースのエラーが起きたときの対処

今日の作業で `Unterminated string in JSON` というエラーが出ました。これは `max_tokens` が小さすぎてAIの返答が途中で切れ、JSONが壊れた状態になったためです。

**対処法**: `max_tokens` を増やす（今回は 1024 → 2048 に変更しました）

本番環境では、万が一JSONが壊れていても安全に対処できるようにすることが大切です。

```typescript
// より安全なJSON解析（AIがJSONブロックを含む場合にも対応）
try {
  const jsonMatch = text.match(/\{[\s\S]*\}/)
  const parsed = JSON.parse(jsonMatch ? jsonMatch[0] : text)
  return parsed
} catch {
  throw new Error('AIのレスポンスをJSONとして解析できませんでした')
}
```

## よくあるミスと対処法

1. **返答が途中で切れる** → `max_tokens` が小さすぎる。特にJSON形式で返してもらう場合は余裕を持った値にする
2. **チャットが文脈を理解しない** → 会話履歴をAIに渡していない可能性がある。毎回の会話ですべての履歴を送る必要がある
3. **APIキーをフロントエンドに書いてしまった** → `.env.local` に書き、変数名に `NEXT_PUBLIC_` をつけないこと
4. **429エラー（リクエスト多すぎ）** → 短時間に大量のリクエストを送るとAPI制限に引っかかる。本番運用では間隔を空ける工夫が必要
5. **コストが予想以上にかかる** → モデルによってコストが大きく違う。Opusは高精度だが高コストなので、用途に合わせて Haiku を活用する
