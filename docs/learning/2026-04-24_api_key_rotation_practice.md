# APIキーを実際に差し替えてみた：学んだこと

## この記事で学べること

- どのキーを差し替える必要があるか（全部ではない）
- Vercel に反映されたか確認する方法
- キーが git 履歴に残っていないか調べる方法

---

## 背景：何が起きたか

Anthropic の API キーが流出した可能性があったため、以下の手順で対応しました。

```
1. Anthropic コンソールで旧キーを削除・新キーを発行
2. .env.local のキーを新しいものに更新
3. Vercel の環境変数を新しいキーに更新
4. Vercel で Redeploy
5. 本番環境でAI機能が動くことを確認
```

---

## 「どのキーを変えるか」を判断する

`.env.local` には複数のキーが入っています。でも、**全部を変える必要はありません**。

### NEXT_PUBLIC_ がついているキーは公開前提

```
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJh...
```

`NEXT_PUBLIC_` というプレフィックス（接頭辞）は Next.js の特別なルールで、**このキーはブラウザに公開しますよ**という意味です。

ブラウザに公開されるということは、誰でも開発者ツールを開けば見られます。
つまり「流出した」という概念が当てはまらないキーです。変える必要はありません。

Supabase の anon key はこの設計を前提にしており、セキュリティは別の仕組み（Row Level Security）で守られています。

### NEXT_PUBLIC_ がついていないキーはサーバーのみで使うシークレット

```
ANTHROPIC_API_KEY=sk-ant-api03-...
```

こちらはブラウザには絶対に公開されないキーです。
サーバー（API Routes）の中だけで使われ、外から見えません。
**漏洩したら悪用されるリスクがある**ため、今回差し替えたのはこちらです。

---

## Vercel に反映されたか確認する方法

Vercel の管理画面で環境変数を更新しただけでは**すぐには反映されません**。
新しい環境変数を使うには、再ビルド（Redeploy）が必要です。

### ステップ

1. Vercel ダッシュボード → プロジェクト → **Deployments** タブを開く
2. 環境変数を変更した**後**に Redeploy が走っているか確認する
3. 本番 URL でアプリを開き、AI 機能を実際に使ってみる

エラーが出ずに AI が応答すれば、新しいキーが正しく動いています。
これが一番確実な確認方法です。

---

## キーが git 履歴に残っていないか調べる

`.env.local` を git に含めていても、もし誰かが誤ってコミットしていたら
**キーが git 履歴に永久に残ってしまいます**（git の履歴は書き換えにくい）。

以下のコマンドで確認できます。

```bash
# .gitignore に含まれているか確認
git check-ignore -v .env.local

# 一度でもコミットされた履歴があるか確認
git log --all --full-history -- .env.local
```

今回の確認結果：
- `.gitignore` の34行目で `.env*` として除外されていた → **問題なし**
- git 履歴には一度もコミットされていなかった → **問題なし**

出力が空なら「コミットされたことがない」という意味です。

---

## .env.local はなぜ gitignore するのか

`.gitignore` に書かれたファイルは git に追跡されません。
`.env.local` には本物のキーが入っているため、絶対に GitHub に上げてはいけません。

Next.js のプロジェクトでは最初から `.env*` がデフォルトで gitignore されています。
**このファイルは自分のパソコンと Vercel の設定画面だけで管理するもの**と覚えておきましょう。

---

## まとめ

| 確認項目 | コマンド・場所 |
|---------|-------------|
| どのキーを変えるか | `NEXT_PUBLIC_` がないもの（サーバー専用キー）だけ変える |
| Vercel に反映されたか | Redeploy 後に本番でAI機能を実際に動かして確認 |
| gitに漏れていないか | `git log --all --full-history -- .env.local` |
| gitignoreされているか | `git check-ignore -v .env.local` |
