# APIキーが漏洩した場合の対応手順

## なぜAPIキーの管理が重要か

APIキーは「パスワード」と同じです。
悪意ある第三者に知られると、あなたのアカウントで勝手にAPIを使われ、
**料金が発生したり、サービスを悪用されたりする**可能性があります。

### 漏洩しやすい場面

- `.env.local` をAIに読ませたときにチャット画面に表示された
- 画面共有中に `.env.local` を開いてしまった
- 誤って GitHub に `.env.local` をコミットしてしまった

---

## 安全な手順：キーの再生成と差し替え

古いキーを削除する**前に**新しいキーを各環境に設定しておくことで、
サービスが一瞬も止まらずに切り替えられます。

### ステップ1: ローカルの `.env.local` を更新

まず手元の環境から先に新しいキーに差し替えます。
この時点ではまだ古いキーは有効なので、アプリは問題なく動き続けます。

```
ANTHROPIC_API_KEY=sk-ant-api03-新しいキー（後で発行）
```

※ 新しいキーはステップ3で発行します。先に場所だけ把握しておきましょう。

### ステップ2: Vercel の環境変数を更新

Vercel ダッシュボード → プロジェクト → **Settings → Environment Variables**
→ `ANTHROPIC_API_KEY` の値を新しいキーに書き換えて保存。

### ステップ3: Anthropic コンソールで新しいキーを発行・古いキーを削除

1. [console.anthropic.com](https://console.anthropic.com) にログイン
2. **API Keys** を開く
3. **Create Key** で新しいキーを発行してコピー
4. ステップ1・2 で貼り付けた値を新しいキーに差し替え
5. 古いキーを **Delete（無効化）**

古いキーを削除した瞬間から無効になります。
ステップ1・2 を済ませてからここに来ることで、切り替えがスムーズです。

### ステップ4: Vercel で Redeploy

Deployments タブ → 最新のデプロイ → **Redeploy**

新しい環境変数を反映させるために再ビルドが必要です。

---

## 手順のまとめ

```
1. .env.local を新しいキーに更新
2. Vercel の環境変数を新しいキーに更新
3. Anthropic コンソールで新しいキーを発行 → 古いキーを削除
4. Vercel で Redeploy
```

**削除は必ず最後**。先に新しいキーを各環境に設定してから削除するのが安全です。

---

## 漏洩を防ぐための日頃の習慣

| やること | 理由 |
|---------|------|
| `.env.local` は `.gitignore` に含める | GitHub に誤ってコミットしない |
| AIに `.env.local` を読ませるときは注意 | チャット画面にキーが表示されることがある |
| キーが漏洩したと思ったらすぐ再生成 | 時間が経つほどリスクが高まる |
| 本番用・開発用でキーを分ける | 漏洩時の被害範囲を最小化できる |
