# API Routesを理解しよう - サーバー側の処理

---

## API Routeって何？

**ひとことで言うと**: ブラウザからの「お願い」を受け取って、データベースやAIとやり取りするサーバー側の処理です。

Next.js では `app/api/xxx/route.ts` というファイルを作るだけで、`/api/xxx` というURLが自動的に作られます。ブラウザから `fetch('/api/xxx')` で呼び出すことができます。

**なぜAPI Routeが必要か**: AIのAPIキーやデータベースの操作はブラウザ側で行うと危険です。APIキーがブラウザに露出してしまうからです。API Routeを使うことで、これらの処理をサーバー側だけで行えます。

---

## このアプリの3つのAPI

### 1. `/api/projects/generate` - 案件を生成する

ユーザーが「案件を生成」ボタンを押したときに呼ばれます。

**処理の流れ**:
```
① ログインしているか確認（未ログインなら「認証が必要」エラーを返す）
② リクエストから難易度（ジュニア/ミドル/シニア）とカテゴリを受け取る
③ Claude AIに案件の内容を生成してもらう
④ 生成された案件をSupabaseに保存する
⑤ 保存した案件データをブラウザに返す
```

**ポイント**: AIのAPIキー（`ANTHROPIC_API_KEY`）はサーバー側にしか存在しません。`NEXT_PUBLIC_` というプレフィックス（頭につく文字）がない環境変数はブラウザに届かないので安全です。

---

### 2. `/api/chat` - AIクライアントとチャットする

ユーザーがチャット欄でメッセージを送ったときに呼ばれます。

**処理の流れ**:
```
① ログインしているか確認
② 「この案件は本当にこのユーザーのものか」を確認（他人の案件に書き込めないようにするため）
③ 案件の情報（AIペルソナ・要件定義）をデータベースから取得
④ これまでのチャット履歴を取得
⑤ ユーザーのメッセージをデータベースに保存
⑥ Claude AIに「あなたはこういうキャラのクライアントです」という設定を与えて返答を生成
⑦ AIの返答もデータベースに保存
⑧ チャット全履歴をブラウザに返す
```

---

### 3. `/api/review` - コードをレビューする

ユーザーがコードを提出したときに呼ばれます。

**処理の流れ**:
```
① ログインしているか確認＋案件の所有者確認
② 提出されたコードをデータベースに保存
③ Claude AIにコードレビューを依頼（可読性・セキュリティ・機能性の3観点でスコアをつける）
④ レビュー結果をデータベースに保存
⑤ 「レビュー済み」というステータスに更新
⑥ スコアが80点以上なら「完了」にして擬似報酬を付与
⑦ レビュー結果をブラウザに返す
```

## エラーの種類と使い分け

APIは処理が成功したか失敗したかを「ステータスコード」という数字で返します。

| コード | 意味 | このアプリでの使われ方 |
|---|---|---|
| `200` | 成功 | データの取得・保存が正常に完了 |
| `401` | 未認証（ログインしていない） | ログインせずにAPIを叩こうとした |
| `403` | 権限なし（ログインしているが許可されていない） | 他人の案件にアクセスしようとした |
| `404` | 見つからない | 存在しないデータにアクセスしようとした |
| `500` | サーバー内部エラー | 予期しないバグや外部サービスの失敗 |

**覚え方**: 4で始まるのは「リクエストした側の問題」、5で始まるのは「サーバー側の問題」です。

## API Routeの基本パターン

実務でよく使われる書き方のテンプレートです。

```typescript
export async function POST(request: NextRequest) {
  try {
    // ステップ1: ログインチェック（していなければ401を返す）
    const { data: { user } } = await supabase.auth.getUser()
    if (!user) return NextResponse.json({ error: '...' }, { status: 401 })

    // ステップ2: 権限チェック（他人のデータでないか確認）
    const { data: record } = await supabase.from('...').select()...
    if (!record) return NextResponse.json({ error: '...' }, { status: 403 })

    // ステップ3: メインの処理（データ取得・AI呼び出しなど）
    const result = await doSomething()

    // ステップ4: 成功したら結果を返す
    return NextResponse.json({ success: true, data: result })

  } catch (err) {
    // ステップ5: 予期しないエラーが起きたら500を返す
    console.error(err)
    return NextResponse.json({ error: '...' }, { status: 500 })
  }
}
```

## よくあるミスと対処法

1. **`await request.json()` を2回呼んでしまった** → リクエストの本文は一度しか読み込めない。変数に入れてから使い回す
2. **環境変数が読めない** → ブラウザで使う環境変数には `NEXT_PUBLIC_` をつける。サーバー専用なら不要
3. **Supabaseのエラーを見落とした** → Supabaseは `throw` でエラーを出さず、`const { data, error }` の `error` に入れて返す。必ず `error` をチェックすること
4. **ステータスコードを指定し忘れた** → `NextResponse.json()` のデフォルトは200。エラーのときは `{ status: 500 }` のように明示的に指定しないと、エラーなのに「成功」と見なされてしまう
